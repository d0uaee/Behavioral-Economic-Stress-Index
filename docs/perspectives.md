# Perspectives

Les perspectives suivantes prolongent naturellement la version finale du
projet, sans contredire les resultats actuels.

## 1. Etendre l'historique cible

La priorite la plus structurante serait d'obtenir une profondeur historique
mensuelle plus large pour l'IPC ou des sous-composantes coherentes.

## 2. Affiner la detection de regime

Pistes possibles :

- tester plusieurs seuils de regime ;
- comparer differents couts de fausse alerte ;
- evaluer des approches de changement de regime explicites.

## 3. Enrichir les signaux comportementaux

Pistes possibles :

- enrichir et stabiliser les sous-indices Trends ;
- tester des signaux plus directement lies aux prix alimentaires ou au
  carburant ;
- mieux documenter la couverture linguistique locale.

## 4. Mieux cibler le macro

Plutot qu'un macro global, une extension plus specifique pourrait porter sur :

- sous-indices alimentaires ;
- energie ;
- transport ;
- composantes directement reliees a l'IPC marocain.

## 5. NLP de seconde generation

L'extension NLP actuelle peut etre amelioree avec :

- sources plus proches des lecteurs ;
- texte integral ou commentaires reellement disponibles ;
- evaluation plus fine de la complementarite avec Trends.

## 6. Industrialisation

Pour une version plus proche de la production :

- tests automatises supplementaires ;
- validation de schemas ;
- automatisation des builds de rapports et de docs ;
- monitoring des changements de source.
